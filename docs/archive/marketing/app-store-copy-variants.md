## Recommendation matrix

| | Variant 1 (Notebook) | Variant 2 (Anti-Bloat) | Variant 3 (Speed-First) |
|---|---|---|---|
| **Core hook** | Familiar metaphor, emotional resonance | Trust through removal, contrarian signal | Concrete measurable claim |
| **Searcher who converts best** | "gym notebook app", "workout log", "training log" searchers | "no account gym app", "simple workout tracker" — emerging intent | "fast gym tracker", "quick set logging" — niche but high-intent |
| **Risk** | Notebook metaphor may undersell the tech advantage | Negative framing can read as defensive if poorly executed | "Under 3 seconds" requires the claim to hold on first use — sets expectation |
| **Brand voice fit** | High — calm, peer-voice, "upgrade" not "revolution" | High — honest, anti-hype, earned trust | Medium — speed claims read as marketing unless immediately verifiable |
| **Discovery keywords served** | gym notebook, training log, workout log, gym tracker | simple gym app, no account workout, minimal gym tracker | fast gym tracker, quick logging, one tap workout |
| **Audience targeting** | Notes-app + paper-notebook switchers (widest pool) | Dropouts from Hevy/Jefit/Fitbod (already frustrated) | Gym-test-aware, speed-sensitive power users |

### Recommended launch variant

**Variant 1 — "Notebook Replacement"**

Reasoning:
- Matches the launch-plan.md hero copy anchor ("notebook replacement for intermediate-to-advanced lifters")
- Widest searcher comprehension — "gym notebook", "training log", "workout log" are established search terms
- Notebook metaphor signals expertise without requiring explanation: the reader instantly understands "it's the thing I already use, but better"
- Anti-bloat (Variant 2) is a strong evergreen but works better once "no AI/no social" is a recognized category in App Store search — it's not yet
- Speed-first (Variant 3) works best once ghost values are confirmed to land as promised under real-gym conditions — validate in TestFlight first

### A/B testing protocol

Apple's Product Page Optimization (PPO) allows up to 3 treatments tested against the default.

**Phase 1 (App Store launch, Weeks 1–4):** Ship Variant 1 as default. Monitor impressions, conversion rate, and install source in App Store Connect.

**Phase 2 (after 500+ impressions, ~Week 4–6):** Create PPO experiment: Variant 1 (control) vs Variant 3 (treatment). Run for minimum 2 weeks / 500 impressions per treatment before reading results. Primary metric: tap-through to product page. Secondary: install conversion from product page.

**Phase 3 (after Phase 2 results):** If Variant 3 wins, run Variant 1 vs Variant 2. If Variant 1 wins Phase 2, run Variant 1 vs Variant 2 to test the anti-bloat angle with the established conversion baseline.

**Subtitle note:** Subtitle is included in App Store Search indexing. Variant 3's subtitle ("Log a set in under 3 seconds.") contains the exact phrase most likely to capture speed-intent search — consider using Variant 3's subtitle even when running Variant 1's description as default. Subtitles cannot be A/B tested separately via PPO, but can be changed during review cycles.

**Do not test:** Variant 2 vs Variant 3 without running either against Variant 1 first. Always anchor against the control.
